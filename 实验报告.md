## 2. FixMatch 算法原理与实现

### 2.1 算法背景

半监督学习（Semi-Supervised Learning, SSL）旨在同时利用少量标注数据 $\mathcal{X} = \{(x_b, y_b)\}_{b=1}^{B}$ 和大量无标注数据 $\mathcal{U} = \{u_b\}_{b=1}^{\mu B}$ 来训练深度学习模型。FixMatch是该领域的重要里程碑，其核心思想极为简洁——**利用模型对弱增强图像的预测作为伪标签，监督同一图像强增强版本的输出**。

### 2.2 损失函数

FixMatch 的总损失由两部分组成：

$$
\mathcal{L} = \mathcal{L}_s + \lambda_u \cdot \mathcal{L}_u
$$

#### 2.2.1 有监督损失 $\mathcal{L}_s$

对有标签数据使用标准的交叉熵损失：

$$
\mathcal{L}_s = \frac{1}{B}\sum_{b=1}^{B} H(y_b, p_{\text{model}}(y \mid \alpha(x_b)))
$$

其中 $\alpha(\cdot)$ 为弱增强（RandomHorizontalFlip + RandomCrop），$H(\cdot,\cdot)$ 为交叉熵函数。

#### 2.2.2 无监督损失 $\mathcal{L}_u$

无监督损失是 FixMatch 的核心创新点，分三步完成：

**Step 1 — 伪标签生成**：对无标签图像施加**弱增强** $\alpha(\cdot)$，用模型预测其类别分布：

$$
q_b = p_{\text{model}}(y \mid \alpha(u_b))
$$

取最大概率类别作为伪标签 $\hat{y}_b = \arg\max(q_b)$。

**Step 2 — 置信度过滤**：仅保留预测置信度高于阈值 $\tau$ 的伪标签。这是 FixMatch 区别于早期伪标签方法的关键——低置信度的伪标签往往是噪声，保留它们会损害训练：

$$
\text{mask}_b = \mathbb{1}[\max(q_b) \geq \tau]
$$

本实验中 $\tau = 0.95$，意味着仅约 5%–50% 的无标签样本参与训练（比例随训练进程增长）。

**Step 3 — 一致性正则化**：对同一无标签图像施加**强增强** $\mathcal{A}(\cdot)$（RandAugment + RandomErasing），用 Step 1 生成的伪标签监督强增强版本的输出：

$$
\mathcal{L}_u = \frac{1}{\mu B}\sum_{b=1}^{\mu B} \text{mask}_b \cdot H(\hat{y}_b, p_{\text{model}}(y \mid \mathcal{A}(u_b)))
$$

强增强迫使模型学习更具鲁棒性的特征表示，而只有高置信度伪标签参与训练则保证了无监督信号的可靠性。"弱增强生成伪标签 + 强增强一致性正则化"的设计，使 FixMatch 以极简的形式达到了当时最优的半监督学习性能。

### 2.3 核心模块的代码实现

根据实验要求，以下核心模块为手动编写：

#### 模块 1：伪标签生成与阈值过滤（`train.py` 第 264–271 行）

```python
# 1. 用弱增强图像生成伪标签（无梯度计算）
with torch.no_grad():
    logits_u_w = model(inputs_u_w)
    max_probs, pseudo_labels = torch.max(
        torch.softmax(logits_u_w, dim=1), dim=1
    )
    # 2. 置信度阈值过滤：仅保留 prob >= 0.95 的样本
    mask = max_probs.ge(args.threshold).float()

# 3. 用伪标签监督强增强版本的输出
logits_u_s = model(inputs_u_s)
loss_u = (F.cross_entropy(logits_u_s, pseudo_labels, reduction='none') * mask).mean()
```

**设计要点**：

- 伪标签生成时不计算梯度（`torch.no_grad()`），防止无监督信号通过伪标签分支反向传播导致模型坍缩。
- 使用 `reduction='none'` 逐样本计算损失后乘以 mask，实现对低置信度样本的精确排除。
- `mask.float().mean()` 可监控伪标签利用率，反映模型对无标签数据的信心。

#### 模块 2：强弱数据增强（`utils/augment.py`）

弱增强 $\alpha(\cdot)$（用于伪标签生成和有标签训练）：

```python
transforms.Compose([
    transforms.RandomHorizontalFlip(),
    transforms.RandomCrop(size=32, padding=4, padding_mode='reflect'),
    transforms.ToTensor(),
    transforms.Normalize(mean=CIFAR10_MEAN, std=CIFAR10_STD)
])
```

强增强 $\mathcal{A}(\cdot)$（仅用于无监督一致性正则化）：

```python
transforms.Compose([
    transforms.RandomHorizontalFlip(),
    transforms.RandomCrop(size=32, padding=4, padding_mode='reflect'),
    transforms.RandAugment(num_ops=2, magnitude=10),  # 核心：随机增强
    transforms.ToTensor(),
    transforms.RandomErasing(p=0.5, scale=(0.02, 0.2)),  # 随机擦除
    transforms.Normalize(mean=CIFAR10_MEAN, std=CIFAR10_STD)
])
```

**设计要点**：

- 强/弱增强的差异化是 FixMatch 成功的关键——弱增强保证伪标签的稳定性，强增强迫使模型学习不变性。
- RandAugment 从 14 种增强操作中随机选择 2 种，以固定强度应用，大幅简化了 AutoAugment 的搜索过程。
- 无标签数据的弱增强和强增强版本均来自同一原始图像，仅在增强管道不同，这确保了语义一致性。

#### 模块 3：EMA 指数移动平均（`train.py` 第 24–78 行）

```python
class EMAModel:
    def __init__(self, model, decay=0.999):
        self.shadow = {}   # 存储参数的指数滑动平均值
        self.decay = decay

    def update(self):
        """在每次 optimizer.step() 后调用"""
        for name, param in self.model.named_parameters():
            if param.requires_grad:
                # shadow = decay * shadow + (1 - decay) * param
                self.shadow[name].mul_(self.decay).add_(param.data, alpha=1 - self.decay)
```

**设计要点**：

- EMA 将模型参数的指数滑动平均作为最终推理参数，有效平滑训练过程中的参数波动。
- 衰减率 0.999 意味着当前参数仅贡献 0.1% 的更新，影子参数更接近训练的长期趋势，泛化性能更好。
- 评估时将影子参数临时加载到模型中，评估完成后恢复原始参数继续训练。

#### 模块 4：数据划分策略（`dataset.py`）

```python
def split_labeled_unlabeled(targets, n_labeled_per_class, num_classes=10):
    """按类别均匀采样有标签样本，所有样本均作为无标签数据"""
    for i in range(num_classes):
        idx = np.where(targets == i)[0]
        sampled = np.random.choice(idx, n_labeled_per_class, replace=False)
        labeled_idx.extend(sampled)
    unlabeled_idx = np.array(range(len(targets)))  # 包含所有训练数据
    return labeled_idx, unlabeled_idx
```

**设计要点**：

- 采用**类别均匀采样**策略，确保标注数据覆盖所有 10 个类别。
- 无标签集包含全部训练数据（含已标注的），但标签信息在无监督分支中不可见。
- 使用固定随机种子（seed=42）确保数据划分可复现。

## 5. 与 USB 库实现的对比

### 5.1 USB（原 TorchSSL）库简介

USB（Unified Semi-supervised learning Benchmark）是微软发布的开源半监督学习基准库（原 TorchSSL 的升级维护版本），提供了 FixMatch、MixMatch、FlexMatch 等多种算法的统一、标准化实现。USB 库的代码质量高、配置完整、实验结果可复现，是半监督学习领域广泛使用的基准工具。

按照实验要求，我尝试使用 USB 库训练 FixMatch。通过查看 USB 配置文件（`usb_official/config/classic_cv/fixmatch/fixmatch_cifar10_*_0.yaml`），USB 的 FixMatch 与自有实现的主要差异如下：

| 差异项                   | 自有实现                | USB 实现                            |
| ------------------------ | ----------------------- | ----------------------------------- |
| **训练迭代数**     | ~262,144 步 (256×1024) | **1,048,576 步** ($2^{20}$) |
| **温度锐化 $T$** | 无（直接 softmax）      | **0.5**（sharpen）            |
| **use_cat**        | 否（分别前向传播）      | 是（拼接前向传播）                  |
| **混合精度 AMP**   | 是 (FP16)               | 否 (FP32)                           |
| **随机种子**       | 1 (seed=42)             | 3 (seed=0/1/2)                      |

**然而**，由于 USB 的 FixMatch 需要训练 **1,048,576 步**——为自有实现的 **4 倍** ，受限于实验条件和作业截止时间，**未能完整运行 USB 库的训练流程**。

因此，本节直接引用 **USB 官方基准测试结果**（`usb_official/results/classic_cv.csv`，3 个随机种子均值 ± 标准差）进行对比分析。

### 5.2 性能对比

USB 官方基准以**错误率**报告，转换为准确率后与自有实现对比如下：

| 标注量 | 自有实现 (Best EMA) | USB FixMatch (官方基准) |   差距   |
| :----: | :-----------------: | :----------------------: | :------: |
|   40   |       69.55%       | **91.67%** ± 1.41 | −22.12% |
|  250  |       91.79%       | **95.03%** ± 0.05 | −3.24% |
|  4000  |       94.62%       | **95.71%** ± 0.10 | −1.09% |

> USB 数据来源：`usb_official/results/classic_cv.csv`。全监督基线（WRN-28-2，50,000 标签）：~95.42%。

**关键发现**：

- **40 标签差距最大（−22.12%）**：USB 在此场景下达到了 91.67%，远超自有实现的 69.55%。
- **性能差距随标注量增加急剧缩小**：250 标签时差距缩至 −3.24%，4000 标签时仅 −1.09%。
- **自有实现与 USB 在 4000 标签场景已非常接近**（94.62% vs 95.71%），说明标注充足时，实现细节的影响大幅减弱。

### 5.3 差距原因分析

#### (1) 训练迭代数 — 最主要原因

USB 使用 $2^{20} = 1{,}048{,}576$ 步（FixMatch 论文标准设置），自有实现仅训练了 $256 \times 1024 \approx 262{,}144$ 步，为 USB 的 **1/4**。

自有实现在约第 100 epoch 后准确率趋于饱和（如 40 标签在 66%–69% 之间停滞），而 USB 的额外 786K 步训练使模型——特别是在 40 标签场景——持续从无标签数据中提取有用信息，最终达到 91.67%。这说明 **FixMatch 的低标注场景需要大量训练迭代来逐步提升伪标签质量**，短训练会严重低估算法潜力。

#### (2) 温度锐化 $T=0.5$ — 重要因素

USB 在伪标签生成时使用温度参数进行锐化：

$$
\text{sharpen}(q)_i = \frac{q_i^{1/T}}{\sum_j q_j^{1/T}}, \quad T=0.5
$$

当 $T < 1$ 时，锐化放大了高置信度类别与低置信度类别之间的差距：

- 使高置信度伪标签更加"确定"，增强训练信号强度
- 帮助更多样本达到阈值 $\tau=0.95$，提高伪标签利用率
- 在低标注场景中，这一点尤为关键——自有实现因缺少锐化，伪标签区分度不足，大量无标签样本被浪费

#### (3) 随机种子与统计可靠性 

USB 报告 3 个种子的均值 ± 标准差，自有实现仅运行 1 个种子（seed=42）。在 40 标签场景中，USB 的标准差达 ±1.41%，说明不同数据划分对结果有显著影响。单次运行可能未捕获最佳划分。

#### (4) 其他因素 

- **use_cat**：USB 拼接三个 batch 做单次前向传播，数学上等价，对准确率几乎无影响。
- **AMP vs FP32**：AMP 对准确率影响通常 < 0.1%，不是差距的主要原因。

### 5.4 改进方向

1. **增加训练迭代至 1M 步**：将 `epochs` 增至约 1024，使总步数达到 $2^{20}$。
2. **引入温度锐化**：在伪标签生成时加入 $T=0.5$：
   ```python
   probs = torch.softmax(logits_u_w / T, dim=1)
   ```
3. **多种子运行**：使用 3 个随机种子，报告均值 ± 标准差。

## 6. FixMatch 与其他半监督算法的比较

### 6.1 半监督学习算法演进

半监督深度学习经历了从早期一致性正则化方法到现代混合方法的演进：

| 方法                   | 年份 | 核心思想                       | 增强策略                      |  伪标签  |
| ---------------------- | :--: | ------------------------------ | ----------------------------- | :------: |
| Π-Model [2]           | 2017 | 同一图像两次前传输出一致       | 基础增强 + Dropout            |    无    |
| Mean Teacher [3]       | 2017 | 教师模型监督学生模型           | 基础增强 + EMA                |    无    |
| **MixMatch** [4] | 2019 | 一致性 + 熵最小化 + MixUp      | K 次弱增强取平均              | 温度锐化 |
| **FixMatch** [5] | 2020 | 弱增强→伪标签，强增强→一致性 | 弱增强 + 强增强 (RandAugment) | 阈值过滤 |

### 6.2 FixMatch 与 MixMatch 详细对比

MixMatch 和 FixMatch 是半监督学习中两个具有代表性的方法，FixMatch 可视为 MixMatch 的"简化+增强"版本。

#### 6.2.1 算法流程对比

**MixMatch 的完整流程**（两步混合策略）：

```
输入: 有标签 batch X, 无标签 batch U
1. 对每个无标签图像 u，生成 K 次弱增强版本（K=2）
2. 对 K 个版本的预测取平均: q̄ = avg(softmax(model(weak_aug(u))))
3. 温度锐化: sharpen(q̄) = q̄^(1/T) / sum(q̄^(1/T))
4. 将有标签 X 和无标签 U（使用锐化伪标签）混合打乱
5. 对整个混合 batch 施加 MixUp（β分布采样 λ，线性插值图像和标签）
6. 对 MixUp 后的有标签部分使用交叉熵损失 L_s
7. 对 MixUp 后的无标签部分使用 MSE 损失 L_u (L2 距离)
8. L_u 的权重从 0 线性 warm-up 至 λ_u
```

**FixMatch 的极简流程**（单步阈值策略）：

```
输入: 有标签 batch X, 无标签 batch U
1. 对每个无标签图像 u，生成 1 次弱增强（预测伪标签）
2. 直接使用 softmax 概率（无温度锐化，无 K 次平均）
3. 仅保留 max(prob) >= 0.95 的高置信度伪标签
4. 对同一样本施加强增强（RandAugment）
5. 用伪标签以交叉熵损失监督强增强版本的输出
6. 总损失 = L_s + L_u（无需 warm-up，固定权重）
```

#### 6.2.2 关键差异总结

| 维度                 | MixMatch                   | FixMatch                                  |
| -------------------- | -------------------------- | ----------------------------------------- |
| **伪标签生成** | K 次增强平均 + 温度锐化    | 单次弱增强 + 直接 softmax                 |
| **置信度策略** | 所有样本参与训练（无过滤） | 高阈值过滤（$\tau=0.95$），低置信度丢弃 |
| **数据增强**   | K 次弱增强 + MixUp         | 弱增强 vs 强增强（RandAugment）           |
| **无监督损失** | MSE（L2 距离）             | 交叉熵（更尖锐的监督信号）                |
| **损失权重**   | 线性 warm-up               | 固定权重$\lambda_u=1$                   |
| **计算复杂度** | 较高（K 次前传 + MixUp）   | 较低（2 次前传，无需 MixUp）              |
| **超参数数量** | 较多（T, α, K, warm-up）  | 较少（$\tau$, $\mu$）                 |

#### 6.2.3 设计差异

**MixMatch** 的设计是"混合一切"：

- 将一致性正则化、熵最小化、MixUp 正则化、温度锐化等多种技术统一融合。
- 通过 MixUp 在标注和无标注样本之间建立平滑插值，创造"虚拟训练样本"，增强模型的插值泛化能力。
- 所有无标签样本都参与训练（K 次平均缓解伪标签噪声），用 MSE 做软监督。

**FixMatch** 的设计哲学是"少即是多"：

- 去掉了 MixMatch 中的 K 次平均、温度锐化、MixUp 和损失 warm-up，仅保留伪标签 + 一致性正则化的最简形式。
- 用更激进的方法保证无监督信号质量：不做 K 次平均、不做软标签，而是直接丢弃低置信度样本。
- 用**强增强**（RandAugment）替代 MixUp 来提供正则化，强增强比 MixUp 更简单且效果更好。
- 从 MixMatch 到 FixMatch 的简化说明：**在半监督学习中，"如何选择可靠的无监督信号"比"如何融合多种正则化技术"更重要**。

#### 6.2.4 性能对比

使用 USB 库在同一框架下运行 FixMatch 和 MixMatch 的结果如下（`classic_cv.csv` 存储的数据）：

| 方法                               |   CIFAR-10 (40 labels)   |  CIFAR-10 (250 labels)  |  CIFAR-10 (4000 labels)  |
| ---------------------------------- | :----------------------: | :----------------------: | :----------------------: |
| MixMatch                           |          29.33%          |          62.72%          |          92.62%          |
| **FixMatch**（USB 官方基准） | **91.67%** ± 1.41 | **95.03%** ± 0.05 | **95.71%** ± 0.10 |
| 全监督                             |          95.40%          |          95.45%          |          95.42%          |

**关键观察**：

- **FixMatch 在所有标注量级别上全面优于 MixMatch**。最显著的差距在 40 标签：FixMatch（91.67%）vs MixMatch（29.33%），差距高达 **62.34%**。
- MixMatch 在 40 标签场景下几乎失效，因为低标注场景下 K 次弱增强平均仍不足以产生可靠的伪标签，而 MixUp 混合了错误伪标签反而"污染"了有监督训练。
- **FixMatch 的置信度阈值策略在低标注场景中展现了决定性优势**：宁愿丢弃 95% 的无标签数据，也要保证剩余 5% 的伪标签是可靠的。
- 在 4000 标签时两者差距缩小，因为足够的标注数据本身就保证了伪标签质量，阈值过滤的优势减弱。

#### 6.2.5 MixMatch 与 FixMatch 的适用场景

| 场景                          | 推荐方法           | 原因                                    |
| ----------------------------- | ------------------ | --------------------------------------- |
| 极度稀缺标注 (< 100 labels)   | **FixMatch** | 置信度阈值过滤能有效排除噪声伪标签      |
| 标注量适中 (100–1000 labels) | **FixMatch** | 简洁高效，超参数少                      |
| 标注量充足 (> 1000 labels)    | 两者皆可           | 伪标签质量高，差异缩小                  |
| 计算资源受限                  | **FixMatch** | 无需 K 次前传和 MixUp，训练更快         |
| 需要软标签/不确定性估计       | MixMatch           | MSE 损失 + 温度锐化提供更丰富的概率信息 |

## 7. 结论

### 7.1 实验总结

本实验基于 PyTorch 手动实现了 FixMatch 半监督图像分类算法，在 CIFAR-10 数据集上评估了三种标注量（40/250/4000）下的分类性能，并与 USB 库实现及 MixMatch 算法进行了对比分析。主要结论如下：

1. **FixMatch 是高效的半监督学习方法**：仅需 40 张标签即可达到 69.55% 的准确率，250 张标签可达 91.79%，4000 张标签可达 94.62%，接近全监督。这验证了"弱增强伪标签 + 强增强一致性正则化 + 高置信度过滤"三件套的有效性。
2. **与 USB 库实现的差距主要由训练迭代数不足和缺少温度锐化导致**：自有实现训练 262K 步（USB 的 1/4），且未使用温度锐化（$T=0.5$）。在 40 标签场景中二者共同导致了 22.12% 的差距；在 4000 标签场景则仅差 1.09%，说明标注充足时实现细节的影响大幅减弱。受限于实验条件，USB 库训练未能完整运行，直接引用了官方基准结果进行对比。
3. **FixMatch 在低标注场景中显著优于 MixMatch**：USB 官方基准中，40 标签下 FixMatch（91.67%）vs MixMatch（29.33%），差距 62.34%。置信度阈值过滤是低标注场景的关键——"宁缺毋滥"的伪标签选择策略远优于 MixMatch 的"全量软监督 + MixUp"策略。


### 7.2 改进方向

- 增加训练迭代至 $2^{20}$ 步，预计可显著提升 40 标签场景性能。
- 引入温度锐化（$T=0.5$）增强伪标签区分度。
- 使用多随机种子运行以提升统计可靠性。
- 探索动态置信度阈值以加速低标注场景收敛。
